<summary><b>🏭 Steel Wizard AI — Industrial AI Copilot for Predictive Maintenance</b> &nbsp;|&nbsp; <code>TypeScript · React · Supabase · AI</code></summary>

<br/>

> **Problem:** Industrial steel manufacturing lacks unified AI-assisted maintenance intelligence — engineers operate reactively, without predictive diagnostics, risk scoring, or automated reporting.

**Solution:** Steel Wizard AI is a full-stack TypeScript platform that combines asset health monitoring, failure prediction, spare management, and an AI maintenance copilot into a single production-deployed system for steel plants.

**Architecture:**
```
Critical Industrial Assets (Supabase PostgreSQL)
              │
              ▼
    ┌─────────────────────────┐
    │  Steel Wizard AI Core   │
    ├──────────┬──────────────┤
    │  Assets  │    Alerts    │
    │  Health  │  Severity    │
    │  Scores  │  Ranking     │
    ├──────────┴──────────────┤
    │   AI Maintenance Wizard │
    │  (NL Interface + RAG)   │
    └─────────────────────────┘
              │
     Recommendations & Reports
```

**Key Features:**
- Natural language interface: diagnosis, root cause analysis, RUL prediction, spare recommendations
- Reliability Command Center: plant health scoring, failure probability tracking
- Alert prioritization engine using severity, business impact, and risk classification
- Automated report generation: daily reliability, shift handover, shutdown planning, executive summaries
- React + TypeScript frontend with ShadCN UI and Tailwind CSS

**Stack:** `TypeScript` `React 19` `Tailwind CSS` `ShadCN UI` `Supabase` `PostgreSQL` `Lovable`

**Engineering Highlights:**
- Modular domain architecture: Assets, Alerts, Spares, Reporting — independently deployable
- AI wizard layered over structured PostgreSQL data using natural language → structured query translation
- Production-deployed with live public demo

<details>
<summary><b>🤖 AI Customer Support Agent — LangGraph Agentic Workflow</b> &nbsp;|&nbsp; <code>Python · LangGraph · FastAPI · OpenRouter</code></summary>

<br/>

> **Problem:** Traditional customer support bots follow rigid decision trees and fail on multi-turn, context-sensitive conversations. Production LLM workflows need structured state management.

**Solution:** A LangGraph-orchestrated AI support agent that handles multi-step conversation workflows, maintains memory across turns, and grounds responses in a company knowledge base via RAG.

**Architecture:**
```
User Query
    │
    ▼
Streamlit UI → FastAPI REST Backend → LangGraph State Machine
                                              │
                              ┌───────────────┼───────────────┐
                              ▼               ▼               ▼
                         Memory Agent   RAG Retrieval   OpenRouter LLM
                              │               │               │
                              └───────────────┴───────────────┘
                                              │
                                      Grounded AI Response
```

**Key Features:**
- LangGraph state machine for structured, multi-turn conversation flow management
- FastAPI REST API with Swagger docs at `/docs` — ready for frontend or system integration
- Separate modules: `customer_graph.py` (graph), `langgraph_agent.py` (agent), `memory_agent.py` (state)
- OpenRouter LLM integration supporting model switching without architecture changes
- Knowledge-base-grounded responses reduce hallucination in support contexts

**Stack:** `Python` `LangChain` `LangGraph` `FastAPI` `Streamlit` `OpenRouter` `Uvicorn`

**Engineering Highlights:**
- Clean separation of concerns: graph layer, agent layer, memory layer, and API layer as distinct modules
- Extensible to RAG, sentiment routing, agent handoffs, and ticketing system integration
- REST contract design (`/chat`, `/history`, `/reset`) matches standard conversational API patterns

<details>
<summary><b>🧠 HireMind AI Recruiter — Intelligent Candidate Discovery Engine</b> &nbsp;|&nbsp; <code>TypeScript · Python · FastAPI · Recharts</code></summary>

<br/>

> **Problem:** Keyword-based resume filters discard strong candidates while surfacing keyword-stuffed profiles. Scaling human screening to 100K candidates is operationally infeasible.

**Solution:** HireMind is a multi-factor, explainable candidate scoring engine that evaluates technical depth, production experience, behavioral signals, and risk indicators — built for the India Runs Hackathon.

**Architecture:**
```
Candidate Dataset (100K Profiles)
              │
              ▼
  ┌───────────────────────────────┐
  │  Candidate Intelligence Engine│
  ├───────┬───────┬────────┬──────┤
  │ Tech  │ Prod  │Behav-  │ Risk │
  │ Fit   │ Exp   │ioral   │ Det. │
  │ (40%) │ (20%) │ (15%)  │      │
  └───────┴───────┴────────┴──────┘
              │
    Ranking Engine → Top-100 Shortlist → CSV Export
```

**Scoring Framework:**

| Factor | Weight | Signal |
|--------|--------|--------|
| Technical Fit | 40% | Skills depth, stack match, seniority |
| Production Experience | 20% | Shipped products, team size, scale |
| Behavioral Signals | 15% | GitHub activity, response rate, profile completeness |
| Experience Fit | 10% | Years, domain alignment |
| Domain Evaluation | 10% | Industry, specialization |
| Availability | 5% | Notice period, location |

**Key Features:**
- Multi-dimensional scoring with fully explainable per-candidate reasoning outputs
- Behavioral intelligence: GitHub activity, recruiter response rate, offer acceptance rate
- Trap detection: penalizes keyword-stuffed or misleading profiles
- React + TypeScript frontend: executive dashboard, candidate explorer, behavioral analysis, comparison views
- Dual implementation: TypeScript ranker and Python/FastAPI ranker using identical scoring formulas

**Stack:** `TypeScript` `React 19` `TanStack Start` `Tailwind CSS` `Recharts` `Python` `FastAPI` `Pandas` `NumPy`

**Engineering Highlights:**
- Vectorised scoring with Pandas/NumPy processes 100K profiles in seconds
- REST API: `/ingest`, `/rank`, `/candidate/{id}`, `/compare`, `/stats`
- Scale-out path documented: Sentence Transformers → FAISS ANN → learning-to-rank → LLM explanations

<br/>

> **Problem:** Driver drowsiness and attention monitoring requires reliable, real-time blink detection from live video without dedicated eye-tracking hardware.

**Solution:** A real-time blink detection system using MediaPipe Face Mesh for 468-point facial landmark tracking and Eye Aspect Ratio (EAR) computation for frame-level blink state classification.

- Live webcam processing with per-frame blink counting and streak detection
- Extensible to drowsiness scoring, PERCLOS metric, and alert trigger integration

**Stack:** `Python` `OpenCV` `MediaPipe`
